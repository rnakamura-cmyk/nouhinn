# 応募スキル（Phase 3：フォーム入力・送信）

## トリガー例
```
応募を実行してください。スキルシフトで。
フォーム送信してください。ハイプロで。
```

## 目的
`_drafts.jsonl` を読み込み、作成済みの応募文をフォームに貼り付けて送信する。
**このフェーズでは応募文を考えない。貼って送るだけ。**

---

## ブラウザルール（最重要）

1. **新しいタブを開かない** - 既に開いているタブを使う
2. **既存のログイン済みセッションを使う** - 新しくURLを開いてログインし直さない
3. 操作する前に、まず現在のタブ一覧を確認する
4. 対象サイトが開いているタブがあれば、そのタブに切り替えて使う
5. 対象サイトのタブがない場合のみ、既存タブの1つでURLを開く

---

## Step 0. パスを取得

`config.json` の `base_dir` を読み込む。空欄の場合はこのファイルのフォルダパスを使う。

---

## Step 1. 対象サイトを特定

指示文からサイトを判断して以下を読み込む：

| 指示に含まれるキーワード | 参照フォルダ |
|------------------------|-------------|
| スキルシフト / skillshift | `{base_dir}/sites/skillshift/` |
| ハイプロ / hipro / HiPro Direct | `{base_dir}/sites/hipro/` |
| チイキズカン / chiikizukan | `{base_dir}/sites/chiikizukan/` |

読み込むファイル：
- `{base_dir}/sites/{サイト名}/config.json`
- `{base_dir}/sites/{サイト名}/SKILL.md`（フォーム操作手順）

---

## Step 2. drafts.jsonl を探す

`{base_dir}/inbox/` から `{サイト名}_*_drafts.jsonl` を探す。

- 複数ある場合は**ファイル名の日時が最新のもの**を使う
- 見つからない場合：
  ```
  {サイト名} の drafts.jsonl が見つかりません。
  先に応募文作成を実行してください。（「応募文を作成してください。{サイト名}で。」）
  ```
  と出力して終了。

---

## Step 3. 応募済みURLを読む（重複防止）

`{base_dir}/applied_urls.txt` を読み込む。存在しない場合は空リスト。

---

## Step 4. 再開判定

同じタイムスタンプの `{サイト名}_{タイムスタンプ}_applied.jsonl` が inbox/ に存在する場合、
そこに記録されたURLをすべて読み取り「今回処理済み」として扱う。

→ 途中クラッシュからの再実行でも重複応募しない。

---

## Step 5. 案件に順番にフォーム送信する

drafts.jsonl の内容を順番に読み込む。

各案件について以下の順で処理する：

### 5-1. スキップ判定

以下のいずれかに該当したらスキップ：
- `status` が "skip" の行
- URLが `applied_urls.txt` にある
- URLが今回の applied.jsonl にある

スキップ時は applied.jsonl に1行書いて次へ：
```
{"action":"append_job", ..., "status":"スキップ", "applied_at":"", "memo":"スキップ理由"}
```

### 5-2. 案件詳細ページを開く

SKILL.md の手順に従い案件詳細ページを開く。以下を取得する：
- 報酬（下限・上限）
- 締切日
- 応募数・契約状況
- 本文（NG条件の最終チェック）

NG条件に該当した場合はスキップ（status を「スキップ」にして記録）。

### 5-3. フォームを開いて応募文を貼り付ける

SKILL.md の手順に従いフォームを開き、drafts.jsonl のテキストをそのまま貼り付ける。

**サイト別フィールドマッピング：**

| サイト | drafts.jsonlのキー | フォームのフィールド |
|--------|-------------------|-------------------|
| スキルシフト | `self_pr` | 自己PR テキストエリア |
| スキルシフト | `motivation` | 志望理由 テキストエリア |
| チイキズカン | `essay` | 「経験がどのように活かせそうですか？」テキストエリア |
| ハイプロ | `comment` | 応募コメント テキストエリア |

添付欄がある場合は `{base_dir}/documents/` 内のファイルを添付。

### 5-4. 送信

SKILL.md に記載された送信フローに従って送信。

### 5-5. 送信直後に即書き込み（ここが最重要）

送信が完了したら**その場で以下を2つ実行**し、次の案件へ進む：

**① applied.jsonl に1行追記**
ファイル名: `{base_dir}/inbox/{サイト名}_{draftsのタイムスタンプ}_applied.jsonl`
```
{"action":"append_job","site":"サイト名","job_title":"案件名","category":"カテゴリ","reward_min":"下限","reward_max":"上限","client":"クライアント名","applications":応募数,"contracts":"契約状況","deadline":"YYYY-MM-DD","url":"案件URL","status":"応募済","applied_at":"今日の日付(YYYY-MM-DD)","result":"","memo":""}
```

**② applied_urls.txt にURLを追記**
`{base_dir}/applied_urls.txt` の末尾にURLを1行追加する。

---

## Step 6. 完了処理

全候補を処理し終えたら：

1. drafts.jsonl を `{base_dir}/processed/` に移動する（candidates.jsonl も残っていれば移動）
2. 件数サマリーを出力：
```
{サイト名}: 応募 X件 / スキップ Y件
applied.jsonl を inbox/ に保存しました。次回の sync でスプシに反映されます。
```

---

## ルール

- **応募文は自分で考えない** - drafts.jsonl に書かれたテキストをそのまま使う
- 1レコードは必ず1行で完結させる（改行を含めない）
- 推測値を入れない。画面から取得した事実のみ書く
- エラーで応募できなかった案件は status を「エラー」、memo にエラー内容を書いて記録する
- `max_applications_per_run` を超えたらそこで停止し、残りは次回へ
