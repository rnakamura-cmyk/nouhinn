# 案件応募スキル（ルーター）

## フェーズ構成

案件応募フローは4フェーズに分かれている。それぞれ独立して実行できる。

| フェーズ | 指示文の例 | 参照ファイル | ブラウザ |
|---------|-----------|------------|---------|
| Phase 1: スクレイピング | 「スクレイピングを実行してください。スキルシフトで。」 | `SCRAPE_SKILL.md` | 要 |
| Phase 2: 応募文作成 | 「応募文を作成してください。スキルシフトで。」 | `DRAFT_SKILL.md` | 不要 |
| Phase 3: フォーム送信 | 「応募を実行してください。スキルシフトで。」 | `APPLY_SKILL.md` | 要 |
| Phase 4: スプシ同期 | 「スプシ同期してください。」 | `SYNC_SKILL.md` / `python sync_sheets.py` | 不要 |

---

## ルーティングルール

### 「スクレイピングを実行」「候補を収集」と言われた場合
→ `SCRAPE_SKILL.md` の手順に従う

### 「応募文を作成」「ドラフトを作成」と言われた場合
→ `DRAFT_SKILL.md` の手順に従う

### 「応募を実行」「フォーム送信」と言われた場合
→ `APPLY_SKILL.md` の手順に従う

### 「スプシ同期」「sync」と言われた場合
→ `python sync_sheets.py` を実行する

### 「案件応募を実行」と言われた場合（一括実行）
→ Phase 1 → Phase 2 → Phase 3 を順番に実行する（非推奨：重くなる）

## Cowork スケジュール設定

各フェーズを時間差で起動する。前フェーズのバトンファイルがなければ自動停止する。

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【チイキズカン】
08:00  スクレイピングを実行してください。チイキズカンで。
08:20  応募文を作成してください。チイキズカンで。
08:40  応募を実行してください。チイキズカンで。

【スキルシフト】
09:00  スクレイピングを実行してください。スキルシフトで。
09:20  応募文を作成してください。スキルシフトで。
09:40  応募を実行してください。スキルシフトで。

【ハイプロ】
10:00  スクレイピングを実行してください。ハイプロで。
10:20  応募文を作成してください。ハイプロで。
10:40  応募を実行してください。ハイプロで。

【スプシ同期（全サイト分まとめて）】
11:00  スプシ同期してください。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**ポイント：**
- Phase 1（SCRAPE）は一覧を読むだけなので20分以内に完了する
- Phase 2（DRAFT）はブラウザ不要なので10分以内に完了する
- Phase 3（APPLY）はフォーム送信のみ（文章生成なし）なので従来より大幅に軽い
- 前フェーズが遅延して次のフェーズでバトンファイルがない場合、そのフェーズは自動停止する（翌日リカバリー）
- applied_urls.txt で重複応募は自動防止

## ファイルの流れ（バトン方式）

```
Phase 1: SCRAPE
  └→ inbox/{site}_{timestamp}_candidates.jsonl   ← バトン①

Phase 2: DRAFT（バトン①がなければ停止）
  └→ inbox/{site}_{timestamp}_drafts.jsonl        ← バトン②

Phase 3: APPLY（バトン②がなければ停止）
  ├→ inbox/{site}_{timestamp}_applied.jsonl       ← バトン③
  └→ applied_urls.txt に追記（即時・1件ごと）

Phase 4: SYNC（バトン③がなければ「処理対象なし」で終了）
  ├→ applied.jsonl をスプシに反映
  └→ processed/ に移動
```

candidates.jsonl と drafts.jsonl は sync_sheets.py の処理対象外（action が "candidate" / "draft" のためスキップされる）。
